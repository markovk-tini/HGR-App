# save-output-flow

Use for Save Locations settings, output-type routing, save-path persistence, and save-destination behavior after screenshots, drawings, recordings, or clips.

## Focus
- one clear source of truth for configured save locations
- safe path validation and persistence
- explicit routing by output type
- clean interaction with follow-up voice save prompts

## Must verify
- drawings, screenshots, recordings, and clips each use the correct default location
- manual path entry works
- browse/select-folder works
- invalid paths do not break the app
- post-action save behavior matches the configured destination or explicit spoken override
